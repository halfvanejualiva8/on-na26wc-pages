# on-na26wc-pages

## 项目简介

这是一个用于归档和发布多个独立 HTML 页面的仓库。  
仓库本身不针对任何特定域名或网站，仅作为静态 HTML 文件的存储与展示空间。  
所有页面内容均保持独立，不涉及商业推广或用户数据收集。

## 目录说明

```
.
├── README.md          # 本说明文件
├── index.html         # 首页（可选）
└── pages/             # 页面归档目录
    ├── example1.html  # 示例页面 1
    └── example2.html  # 示例页面 2
```

- `pages/`：存放所有独立 HTML 页面的主目录，可根据需要添加子文件夹。
- `index.html`：可选的首页文件，用于提供导航或索引功能。

## 页面归档说明

- 每个 HTML 页面均为独立文件，不依赖外部资源（除非在页面内显式引用）。
- 页面内容由提交者自行维护，仓库仅负责存档与版本管理。
- 页面命名建议使用有意义但不过长的英文或数字组合，避免特殊字符。
- 如需新增页面，请直接在 `pages/` 目录下创建并提交。

## 维护说明

- 本仓库采用常规 Git 工作流，所有变更通过 Pull Request 或直接提交完成。
- 欢迎提交新的独立页面，但请确保内容不违反所在平台的服务条款。
- 如有大规模修改或重构，建议先通过 Issue 进行讨论。
- 仓库不保证对页面内容进行审核或修改，仅作为存储与发布的中立载体。
